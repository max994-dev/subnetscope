# Subnet 72 — StreetVision (NATIX)

> Analyzed: 2026-05-04 · easy-entry score at time of analysis: 64.8/100

## TL;DR

Subnet 72 is **NATIX**'s roadwork-detection subnet — miners run a binary
image classifier (roadwork yes/no) on street-level photos. The reward
mechanism is the friendliest on the recommended list: top-1 share is
only ~1% (very flat), the project is mature with a real product, and
the model lives on HuggingFace so you don't even need to expose an
inference endpoint. The catch: all 256 slots are full, burn fee is
mid-tier (~0.12 τ), and your model decays after 90 days unless you
re-submit.

## What it actually does

NATIX builds a decentralized street-view dataset for autonomous driving
and HD-map maintenance. The current mining task is **roadwork
detection**:

- Validator pulls a balanced set of real + synthetic street images
- For each image, the validator queries each miner's HuggingFace model
- Miner returns a single float in [0, 1] (>0.5 = roadwork present)
- Score = accuracy + AUC across the test mix, smoothed over a window

You don't host a live axon for inference — you just **publish a model
to HuggingFace** and the validators pull it. That's a huge operational
simplification compared to most subnets.

## Reward structure

Top-1 share of ~1% means emission is spread very evenly across the 256
miners; this is one of the most decentralized subnets on the network.
The gradient comes from accuracy, not from being first. Bigger /
better-tuned models earn more, but no single miner dominates.

| Knob | Effect |
|---|---|
| Model accuracy on validator test set | dominant reward signal |
| Model freshness (90-day decay) | reward factor decays from 1.0 → 0 if you don't re-submit |
| Network throughput | minor; HF hosts the weights |

## What miners run

- **Hardware:** CPU is enough for *running* the miner (the model is on
  HF). A GPU is useful for *training / fine-tuning* a stronger
  classifier locally before pushing it.
- **Stack:** Python axon + the NATIX `run_neuron.py` script that
  registers your HF model id with the validator. Vision model can be
  ResNet-50 fine-tuned, EfficientNet, or anything that fits in HF's
  free quota.
- **External:** HuggingFace account (free tier works), public model
  repo, ~5–50 MB weights.

## Realistic earnings

At today's emission of ~36 τ/day to the subnet across 256 miners:

| Outcome | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| Default base model, no tuning | ~0.05–0.10 τ | $12–$25 |
| Custom fine-tune, mid-pack | ~0.15–0.25 τ | $35–$60 |
| Strong tuned model, top-30 | ~0.3–0.5 τ | $75–$125 |
| Top-1 (rare, lots of effort) | ~0.5–0.8 τ | $125–$200 |

The flat reward shape means there's no "Arena tier" lottery — you get
roughly what you earn.

## Honest assessment

**Pros:**

- Best decentralization on the recommended list (top-1 ~1%)
- Real product, real funding (NATIX is a public network with a token
  outside Bittensor)
- HuggingFace-hosted model = no axon to manage, no uptime risk
- Active maintenance (commits within last 30 days)

**Cons:**

- All 256 slots full → buy out an existing UID (eviction war on the
  weakest miners)
- 0.12 τ burn fee is mid-tier (not the cheapest)
- 90-day reward decay means it's not "set and forget" — you must
  re-submit periodically
- GPU helpful for *building* a competitive model even if inference is
  cheap

## Getting started

1. Read the official miner guide:
   <https://github.com/natixnetwork/streetvision-subnet/blob/main/docs/miner.md>
2. Wallet: hot+cold pair, fund with ~0.15 τ buffer for registration
3. Train or fine-tune a roadwork classifier locally — good starting
   points: pretrained ImageNet model + LoRA on a small open
   street-image dataset
4. Push the model to HuggingFace as a public repo
5. Run `run_neuron.py` against the public testnet validator first to
   confirm your model scores above the median
6. `btcli subnet register --netuid 72 --wallet.name <cold> --wallet.hotkey <hot>`
7. Watch your score for ~24 h; iterate on the model if you're below the
   median
