# Subnet 32 — ItsAI (AI Content Detection)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 41.7/100

## TL;DR

ItsAI (by Manifold) detects **AI-generated text** with ~85% accuracy. Miners
run classifiers that label text as human-written or AI-generated. The
validation mechanism is clever: validators deliberately modify a database of
18M human texts to prevent miners from simply memorizing known answers. Near-
zero burn fee (~0.0005 τ), 143 active miners, but listed as "heavy GPU need"
which likely refers to running competitive BERT/RoBERTa-scale discriminators.

## What it actually does

Validators issue text samples (human or AI-generated) and miners return a
binary classification + confidence score. To prevent gaming:

- Validators use a database of 18M human-authored texts as ground truth
- They **modify** these texts slightly each cycle, so miners can't cache the
  answers
- The "known truths" approach (vs reward model scoring) means miners must
  generalize, not memorize

Applications: detecting student plagiarism, filtering spam, identifying
fraudulent job applications, detecting scam emails.

## Reward structure

Proportional to classification accuracy and F1 score across validator test
batches. Top-1 share is moderate (not winner-take-all). 143 miners split
~36 τ/day. The validation design (modified texts) prevents any single miner
from gaming by memorizing.

## What miners run

- **Hardware:** Heavy GPU is listed but not strictly required for a competitive
  baseline. A fine-tuned RoBERTa-large (1.4B params) needs ~6–8 GB VRAM.
  For top performance: A100/H100 class.
- **Stack:** Python, HuggingFace transformers, the ItsAI repo.
  Fine-tune a discriminator (RoBERTa, DeBERTa, or custom) on a mix of human
  and GPT-4/Claude-generated text.
- **Data:** The RAID benchmark, HC3 dataset, and M4 dataset are good
  fine-tuning sources.

## Honest assessment

**Pros:**
- Near-zero burn fee
- Real-world application with commercial demand
- Clever validation design prevents gaming
- Academic literature rich (AI detection is a hot research area)

**Cons:**
- 143 miners = moderately crowded
- Competitive models need GPU
- Arms race with LLMs: as LLMs improve, detection gets harder (cat-and-mouse)
- "Heavy GPU" listing may mean top competitors use A100+ class hardware

## Getting started

1. SubnetAlpha: <https://subnetalpha.ai/subnet/its-ai/>
2. Wallet + ~0.005 τ buffer
3. Fine-tune a RoBERTa-large on a public AI-detection dataset (RAID benchmark
   is free and well-maintained)
4. Start with CPU inference — verify accuracy ≥ 85% before adding GPU
5. `btcli subnet register --netuid 32 --wallet.name <cold> --wallet.hotkey <hot>`
