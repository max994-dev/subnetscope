# Subnet 66 — Ninja (Coding Agent)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 39.5/100

## TL;DR

Ninja is a **coding agent distillation subnet** — miners solve coding tasks
(generate, debug, refactor code) and validators score solution correctness and
efficiency via CLI. Low GPU need, mid burn fee (~0.19 τ), but only **3 active
miners** with 0 free slots. The tiny miner count is the headline: if the subnet
is healthy, earnings could be exceptional. If it's struggling, you'd be
burning the registration fee on a dormant subnet. Verify activity first.

## What it actually does

Validators issue coding challenges (problem statements, existing code for
debugging, refactoring tasks). Miners return solutions. Validators run the
submissions against test cases and score on:
- Correctness (does it pass tests?)
- Efficiency (runtime / memory complexity)
- Code quality (style, maintainability)

The "agent distillation" framing means the subnet is building a dataset of
human/AI-solved coding tasks that can be used to fine-tune smaller, faster
coding models. The Arena competition format (mentioned in project-nobi docs)
pits miners head-to-head on the same problem with direct comparative scoring.

## Reward structure

Proportional to score. With only 3 active miners sharing ~36 τ/day, average
per-miner emission is ~12 τ/day. Verify this is real before relying on it.

## What miners run

- **Hardware:** Low GPU (the scoring is code execution, not model inference).
  A good CPU is enough. GPU helps if you use a local LLM (Code Llama, Qwen
  Coder) for code generation.
- **Stack:** Python, the ninja repo, CLI axon. Submit code solutions via the
  registered axon endpoint.

## Realistic earnings

With only 3 active miners:

| Setup | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| Working implementation (top-3) | ~5–12 τ | $1250–$3000 |

...but **only if the subnet is actually healthy** with active validators and
bouncing on-chain weights. SubnetAIQ shows the miner count has been very low
for a while — this might reflect a subnet that's not ready for general mining.

## Honest assessment

**Pros:**
- Theoretical earnings are exceptional with only 3 miners
- Low GPU requirement
- Interesting problem (coding agent quality is a core ML research area)

**Cons:**
- 3 active miners suggests the subnet may be poorly-known or struggling
- 0.19 τ burn fee wasted if validators aren't scoring
- All slots technically used; can't join without evicting someone
- Scant public documentation and community

## Getting started

1. SubnetAIQ: <https://subnetaiq.io/subnet/66>
2. **Before registering:** Check on-chain metagraph — are validators updating
   weights regularly? Ask in Bittensor Discord `#subnet-66` if it exists.
3. Wallet + ~0.22 τ buffer
4. If active: implement a Code Llama-based solution generator and test against
   sample challenges in the repo
5. `btcli subnet register --netuid 66 --wallet.name <cold> --wallet.hotkey <hot>`
