# Subnet 105 — Beam (Decentralized Bandwidth / CDN)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 38.8/100

## TL;DR

Beam is a **Proof-of-Bandwidth network** — miners run bandwidth nodes that
route and stream data across the Bittensor ecosystem, earning rewards based on
actual measured throughput, latency, and delivery success. 847 Gb/s global
capacity. Mid burn fee (~0.053 τ), 28 active miners with 0 free slots. If
you have a fast server with good bandwidth but no expensive GPU, this is one
of the few subnets that rewards network infrastructure rather than AI compute.

## What it actually does

Miners run two processes:

1. **Orchestrator** — routing, load balancing, managing which worker handles
   which transfer chunk
2. **Workers** — physically stream data chunks to destinations and generate
   cryptographic Proof-of-Bandwidth records for each completed transfer

Validators verify proofs and score miners on:
- **Throughput** — actual Mb/s delivered
- **Latency** — time to first byte and p99 completion time
- **Delivery success rate** — what fraction of transfers complete without errors

Rewards are distributed in proportion to weighted performance scores, following
standard Yuma Consensus each block.

## Reward structure

Proportional to validated bandwidth performance. No winner-take-all. 28
active miners sharing ~36 τ/day — moderate per-miner emission with relatively
low competition for an infrastructure subnet.

## What miners run

- **Hardware:** No GPU. High-bandwidth VPS or dedicated server. The more
  bandwidth available (both upload and download), the more you earn.
  1 Gbps+ symmetric recommended; 10 Gbps is competitive.
- **Stack:** Python, the Beam node software (`github.com/...`), stable public
  IP, open ports for orchestrator and workers.
- **Key metric:** Sustained throughput at low latency — burst capacity
  doesn't help, consistent delivery does.

## Realistic earnings

At ~36 τ/day across 28 miners:

| Bandwidth | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| 100 Mbps uplink (basic VPS) | ~0.3–0.6 τ | $75–$150 |
| 1 Gbps uplink | ~0.8–1.5 τ | $200–$375 |
| 10 Gbps+ (datacenter) | ~2–4 τ | $500–$1000 |

Subtract VPS/datacenter bandwidth costs from these figures.

## Honest assessment

**Pros:**
- No GPU — this is a bandwidth/infrastructure play
- 28 active miners → moderate competition, decent per-miner emission
- Bandwidth infrastructure has real demand as Bittensor ecosystem grows
- Not an AI modeling problem — different competitive landscape

**Cons:**
- 0.053 τ burn fee is minor but non-zero
- VPS with fast bandwidth costs money (offsets earnings)
- 0 free slots — must evict (but only 28 miners, weakest can be displaced)
- Health rank #67/128 — some validator coverage concerns

## Getting started

1. Website: <https://subnetalpha.ai/subnet/beam/>
2. SubnetRadar: <https://subnetradar.com/subnet/105>
3. Wallet + ~0.06 τ buffer
4. Provision a VPS with 1+ Gbps bandwidth (Hetzner, OVH, Vultr have good
   bandwidth-to-cost ratios)
5. Install Beam node software and configure orchestrator + worker processes
6. Verify your throughput with the validator endpoint before registering
7. `btcli subnet register --netuid 105 --wallet.name <cold> --wallet.hotkey <hot>`
