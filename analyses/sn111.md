# Subnet 111 — oneoneone (Web Review Data)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 48.7/100

## TL;DR

Oneoneone scrapes and enriches **user reviews from across the web** (Google
Maps, Yelp, forums, social media) — miners collect authentic user content,
run NLP enrichment (sentiment, intent, spam detection, translation), and the
validated data is sold via API on oneoneone.io with revenue redistributed to
miners. No GPU required, near-zero burn fee (~0.0006 τ). 249 active miners is
the most crowded on the recommended list. Revenue-sharing model is the novel
angle.

## What it actually does

Mining pipeline:

1. **Scrape** — miners crawl sources (Google Maps, Yelp, blogs, Reddit,
   social media) for user review content using configured scraping rules
2. **Enrich** — each piece of content is run through NLP models:
   - Spam/bot detection
   - User intent classification (complaint, praise, question, suggestion)
   - Sentiment and emotion analysis
   - Multi-language translation
3. **Submit** — enriched records are submitted to validators every 20-30 min
4. **Validate** — validators re-fetch a sample to verify authenticity, then
   score on completeness, freshness, and accuracy of NLP tags
5. **Revenue loop** — verified data is sold via oneoneone.io API; subscription
   revenue is redistributed to miners/validators through TAO-based rewards

## Reward structure

Proportional to score. Freshness (recently published reviews) is rewarded.
Volume matters but uniqueness vs other miners is also weighted. The
revenue-sharing component adds a real-world revenue stream on top of pure
TAO emission — if the API business scales, miner income grows too.

## What miners run

- **Hardware:** CPU only (scraping + NLP enrichment with small models).
- **Stack:** Python, scraping libraries, small NLP models (can use APIs or
  local models). The oneoneone GitHub has the mining framework.
- **Key challenge:** Avoiding rate-limiting/blocking from Google Maps/Yelp.
  Proxies or rotation strategies are part of the competitive edge.

## Realistic earnings

At ~36 τ/day across 249 miners (most crowded of any recommended subnet):

| Outcome | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| Basic single-source scraper | ~0.03–0.08 τ | $7–$20 |
| Multi-source + good NLP enrichment | ~0.10–0.20 τ | $25–$50 |
| Top-50 (high volume + accuracy) | ~0.25–0.4 τ | $62–$100 |

Plus potential revenue-sharing upside if the API product grows.

## Honest assessment

**Pros:**
- Near-zero burn fee
- No GPU required
- Revenue-sharing model creates real-world commercial upside beyond TAO
- Novel subnet concept — data monetization with transparent redistribution

**Cons:**
- 249 miners = most crowded recommended subnet
- Scraping legality is a grey area (Google Maps ToS forbids scraping)
- Scraper arms race — need proxy rotation, rate limiting workarounds
- Small team / early-stage product — commercial traction unclear

## Getting started

1. GitHub: <https://github.com/oneoneone-io/subnet-111>
2. Wallet + ~0.005 τ buffer
3. Start with a single source (Yelp is more scraper-friendly than Google Maps)
4. Add NLP enrichment: spaCy + a translation model for non-English content
5. Test locally for a few hours, check your submission quality against the
   validation endpoint before registering
6. `btcli subnet register --netuid 111 --wallet.name <cold> --wallet.hotkey <hot>`
