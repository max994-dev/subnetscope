# Subnet 47 — EvolAI (LLM Prompt Compression)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 45.0/100

## TL;DR

EvolAI focuses on **lossless text compression for LLM prompts** — miners
develop algorithms that compress input context before sending to language
models, reducing token usage without losing information. No GPU required for
the compression task itself. Burn fee low (~0.05 τ), 194 free UID slots
(one of the only recommended subnets with open slots — no eviction needed).
Only 24 active miners makes per-miner emission high, but the subnet health
rank is poor (#125/128) so research before committing.

## What it actually does

Validators send miners text prompts / context windows and challenge them to:
1. Compress the context to a target compression ratio (e.g., 70% of original
   token count)
2. Return the compressed version
3. Validator decompresses (or runs a task with the compressed prompt) and
   verifies that the decompressed result is semantically equivalent to using
   the full original

Scoring: semantic preservation + compression ratio. Higher compression at
same semantic fidelity = better score.

**Note on project identity:** Some sources list SN47 as "Reboot" (a robotics
subnet) — the on-chain metadata shows EvolAI/compression. Check Discord before
investing significant time, as subnet focus can change.

## Reward structure

Proportional to score. With only 24 active miners and 194 free slots,
competition is very low and per-miner emission is high. However, poor health
rank suggests the subnet may have had instability or low validator activity.

## What miners run

- **Hardware:** CPU only (text compression is CPU-bound).
- **Stack:** Python, NLP libraries (tokenizers, sentence transformers for
  semantic similarity scoring). Custom compression algorithms — LLM-based
  summarization + lossless encoding is the competitive approach.
- **Research angle:** LLM prompt compression is an active research area
  (LLMLingua, AutoCompressors, etc.) — academic papers give a real edge.
