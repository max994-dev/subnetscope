# Subnet 50 — Synth (Mode Network)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 54.7/100

## TL;DR

Synth is a **probabilistic price-path forecasting subnet** — miners generate
Monte Carlo simulations of crypto/forex/equity prices and are scored on
calibration (CRPS), not on picking winners. No GPU required; the quality of
your statistical model matters. All 256 slots full, burn fee high (~0.80 τ),
and the reward shape rewards rolling 10-day consistency so new entrants ramp
slowly. Best for quants and statistics-savvy miners.

## What it actually does

Validators send hourly requests specifying an asset (e.g. BTC, ETH, gold),
a start timestamp, and a required number of simulated price paths (currently
1,000 per request). Miners return a matrix of paths — each path is a
sequence of prices at 5-minute increments over a 24-hour horizon.

Scoring uses **CRPS** (Continuous Ranked Probability Score), the gold standard
for probabilistic forecasts: it rewards both accuracy *and* calibration (a
model that says "I'm 90% sure" and is right 90% of the time beats one that
always predicts a narrow band).

Emissions are split equally between two competitions: 24-hour and 1-hour
high-frequency forecasting.

## Reward structure

Validators compute CRPS scores per miner per request, then:
1. Cap the worst 10% of scores to the 90th-percentile value
2. Subtract the best score from all results (normalizes to "how much worse
   than the best am I?")
3. Apply exponential decay rolling average over 10 days (recent performance
   weighted more)
4. Allocate emissions proportionally to the inverted score (lower raw error =
   higher share)

Top-1 share is moderate (not winner-take-all). Consistent good forecasting
beats single-day lucky runs.

## What miners run

- **Hardware:** CPU only; no GPU needed. Monte Carlo simulation is
  embarrassingly parallel — the more CPU cores, the faster you can generate
  1,000 paths per request within the time window.
- **Stack:** Python, scipy/numpy for simulation, PM2 for process management.
  You implement the `BaseModel` interface in the repo and `run_neuron.py`
  handles the axon.
- **Key files:** `miner_tutorial.md` in the GitHub repo.

## Honest assessment

**Pros:**
- No GPU required
- Real academic problem (CRPS-based scoring is rigorous)
- Consistent model design beats ephemeral hot streaks — rewards skill

**Cons:**
- Burn fee 0.80 τ is the highest on the recommended list
- All 256 slots full — must evict an existing UID
- 10-day rolling average means it takes 10+ days to reflect improved models
- No instant feedback loop — you send a job, wait for the next scoring cycle

## Getting started

1. Clone: `git clone https://github.com/synthdataco/synth-subnet`
2. Read `docs/miner_tutorial.md`
3. Wallet + ~0.90 τ buffer (registration + fees)
4. Implement a simple GBM baseline first to confirm your axon works
5. Iterate: add GARCH, regime-switching, or ensemble approaches for CRPS gains
6. `btcli subnet register --netuid 50 --wallet.name <cold> --wallet.hotkey <hot>`
7. Monitor 10-day rolling score; upgrade model when you see your rank stagnate
