# Subnet 64 — Chutes (Serverless GPU Inference)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 53.2/100

## TL;DR

Chutes is the **leading decentralized serverless AI compute platform** on
Bittensor — market cap rank #1 among all subnets. Miners provide GPU machines
that run open-source models on demand. Burn fee is near-zero (~0.0005 τ), but
you need real GPU hardware. Only 29 active miners out of 256 slots means
emissions per miner are very high right now, but the task is serious
infrastructure work (Kubernetes, CUDA validation, cold-start optimization).

## What it actually does

Users deploy open-source models (Llama, Stable Diffusion, Whisper, etc.) to
Chutes and call them via API. Miners provide the GPU nodes that execute these
deployments. The network is:

- **Compute marketplace:** GPU providers (miners) ↔ model deployers ↔ end users
- **Kubernetes-native:** Miners run K3s (lightweight Kubernetes), with `gepetto.py`
  handling deployment orchestration and cost optimization
- **Hardware-verified:** GraVal (a C/CUDA library) authenticates GPU VRAM and
  compute capacity — you cannot fake an H100 with a 3090
- **Incentive window:** Rewards are calculated on 7-day compute totals —
  consistent availability beats bursty capacity

## Reward structure

Miners earn proportional to verified compute delivered over 7 days. Consistent,
high-uptime nodes earn more than nodes that spike and go offline. Only 29
active miners at time of analysis — per-miner emission is very high (split of
~36 τ/day among 29 = ~1.2 τ/day per miner on average).

## What miners run

- **Hardware:** NVIDIA GPU required. Range from A10 / T4 (cheaper, faster cold
  start) to 8× H100 nodes (maximum earnings). Multiple providers (RunPod,
  Lambda, self-hosted) are used in practice.
- **Stack:** Ubuntu, K3s (Kubernetes), Docker, NVIDIA container toolkit,
  PostgreSQL, Redis, WireGuard VPN (for spanning multiple providers).
- **Orchestration:** `gepetto.py` automates deployment management.
- **Storage:** 100 GB+ SSD per node for model weights.

## Realistic earnings

With 29 active miners sharing ~36 τ/day subnet emission:

| Hardware | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| 1× A10 / T4 | ~0.5–1.0 τ | $125–$250 |
| 1× RTX 4090 | ~0.8–1.5 τ | $200–$375 |
| 1× A100 80GB | ~1.5–2.5 τ | $375–$625 |
| 8× H100 | ~4–8 τ | $1000–$2000 |

Note: if more miners join (the slot capacity is 256), per-miner earnings drop
proportionally. Monitor the active miner count.

## Honest assessment

**Pros:**
- Near-zero burn fee (~0.0005 τ)
- Market cap #1 on Bittensor — large validator ecosystem, serious product
- Only 29 active miners right now → very high per-miner emission
- Real revenue from actual users (not just synthetic validator benchmarks)

**Cons:**
- Requires physical GPU hardware — not a software-only subnet
- Kubernetes/Docker infrastructure complexity
- GraVal hardware verification means you can't fake hardware
- 7-day rolling window means it takes a week to fully ramp up earnings

## Getting started

1. Read: <https://chutes.ai/docs/miner-resources/overview>
2. GitHub: <https://github.com/chutesai/chutes>
3. Prepare a GPU machine with at least 1 NVIDIA GPU and 100 GB SSD
4. Install K3s, Docker, NVIDIA container toolkit
5. Configure WireGuard + set up PostgreSQL + Redis
6. Run `gepetto.py` to register your executor with the network
7. `btcli subnet register --netuid 64 --wallet.name <cold> --wallet.hotkey <hot>`
