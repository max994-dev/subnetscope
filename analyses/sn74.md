# Subnet 74 — Gittensor (Open Source Contributions)

> Analyzed: 2026-05-05 · easy-entry score at time of analysis: 45.6/100

## TL;DR

Gittensor pays miners in TAO for **real, merged GitHub pull requests** to
whitelisted open-source repositories. No model hosting, no GPU, no axon to
maintain — just submit a GitHub PAT and merge PRs. Burn fee is mid (~0.15 τ),
only 16 active miners right now, scoring runs every 2 hours after a PR merges.
If you're already contributing to open source, this is the easiest possible
Bittensor subnet.

## What it actually does

Validators poll whitelisted GitHub repos every 2 hours. When a PR you authored
merges, the validator:

1. Authenticates you as the PR author via your registered PAT
2. Runs **semantic code analysis** to score the contribution:
   - Code quality metrics
   - Repository weight (popular repos score higher)
   - Programming language factor (Rust/Go score higher than, say, docs-only)
3. Assigns rewards proportional to contribution score

Only **merged PRs** count — draft PRs, closed-without-merge, and bot commits
are filtered out. GitHub verification makes Sybil resistance strong — you
can't fake a merge.

## Reward structure

Proportional to contribution scores aggregated over time. No winner-take-all.
Validators rescore every 2 hours after a merge. With only 16 active miners
the per-miner emission is very high.

## What miners run

- **Hardware:** Nothing — no server needed. Your GitHub account is the "miner".
- **Registration:** `btcli subnet register`, then submit your GitHub fine-grained
  PAT via one CLI command — no docker, no PM2, no axon.
- **Ongoing work:** Write and submit PRs to whitelisted open-source repos.
  The validator handles everything else.
- **Competitive edge:** Technical depth (complex code PRs score higher than
  README fixes), popular repos, Rust/Go/C++ vs Python.

## Realistic earnings

With only 16 active miners sharing ~36 τ/day:

| Output | TAO/day | USD/day (TAO ≈ $250) |
|---|---|---|
| 1–2 docs/minor PRs per week | ~0.2–0.5 τ | $50–$125 |
| 3–5 substantive code PRs/week | ~0.8–1.5 τ | $200–$375 |
| Core contributor to major OSS project | ~2–4 τ | $500–$1000 |

These will drop as more miners discover the subnet.

## Honest assessment

**Pros:**
- Zero infrastructure — no server, no GPU, no axon
- Already-productive work (if you contribute to OSS anyway, this is free money)
- Only 16 active miners → exceptional per-miner emission right now
- Strong Sybil resistance; cheating is hard

**Cons:**
- 0.15 τ burn fee is non-trivial given it takes time to ramp earnings
- Whitelisted repos only — you can't pick any project
- Requires genuine coding skill; low-quality PRs get rejected by maintainers
  and score zero
- Repository whitelist is controlled by the subnet owner — unclear governance

## Getting started

1. GitHub: <https://github.com/entrius/gittensor>
2. Miner docs: <https://docs.gittensor.io/miner.html>
3. Wallet + ~0.18 τ buffer
4. Get a GitHub fine-grained PAT with read-only access
5. Check the whitelist of repos in the validator config
6. `btcli subnet register --netuid 74 --wallet.name <cold> --wallet.hotkey <hot>`
7. Register your PAT with the validator via the provided CLI command
8. Start submitting PRs — rewards accrue within 2 hours of each merge
